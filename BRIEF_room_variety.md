# 맵 구조 재설계 v2 + 특수 방 다양화 — 외부 클로드 의뢰용 브리프

> 이전 의뢰(`BRIEF_map_redesign.md` → `DESIGN_map_layout.md`)는 11맵의 platform 좌표만 손봤음.
> 그 결과 **세계 자체는 모두 동일** (4400×720 가로 스크롤 + 우측 골) — 결국 같은 박스 안에서 platform 모양만 바꾼 꼴이라 단조로움 그대로.
>
> 이번 의뢰는 **맵의 세계 구조 자체**를 바꾸는 단계. + 특수 방(보스/도전/이스터에그) 다양화.

## 0. 게임 컨텍스트 (재확인)

- Godot 4.6 횡스크롤 액션 + 로그라이트
- 1회 플레이 8~15분, 부스/QR 단발 플레이
- 7스테이지 선형 진행, 각 stage 진입 전 11맵 풀에서 1픽
- 적 5종, 스킬 트리 3계열×3티어, 결말 4종
- 자세한 컨텍스트: `STORY.md`, `DESIGN_growth_system.md`, `PRD.md`

**중요: 풀 메트로배니아는 컨셉 충돌**. 단발 플레이라 거대 단일 맵·backtrack·능력 게이팅 못 씀. 그러나 **맵 단위로 형태가 다르면** 부스 단발 플레이에서도 매 맵이 새로운 인상.

## 1. 현재 한계 (이번 작업으로 풀어야 할 것)

### 1.1 모든 맵이 같은 세계 박스

```
Stage.gd 하드코딩:
  STAGE_LENGTH = 4400  (가로)
  GROUND_Y = 600       (지면 높이)
  PLAYER_START = (140, 540)   (좌측 시작)
  goal_position = (4320, 540)  (우측 골)
  camera limits: (0, -200) ~ (4400, 800)
```

11개 맵 전부 이 동일한 4400×1000 박스 안에서 platform 좌표만 다르게 두는 구조. 결과:
- 옥상도 가로 4400 진행, 핵심부도 가로 4400 진행, 비상 탈출로도 가로 4400 진행
- 진행 방향 항상 **좌→우**
- 목표 항상 **우측 끝 골**
- 카메라 항상 같은 비율로 가로 follow

## 3. 결정해야 할 것 (외부 클로드와 합의)

### 3.1 맵의 세계 형태 — 어디까지 다르게 할 것인가

**A안 — 완전 자유 형태**
- 11개 맵 모두 고유 세계 크기/방향 정의
- 단점: Stage.gd 대규모 리팩터, 카메라 / goal / spawn 로직 전부 일반화 필요
- 작업량: 매우 큼 (1주+)

**B안 — 3~4가지 템플릿**
- 세계 형태를 정형화: `HORIZONTAL_LONG`, `VERTICAL_UP`, `VERTICAL_DOWN`, `SINGLE_SCREEN_ARENA`
- 각 맵이 이 중 하나를 선택. platform 좌표는 그 안에서 자유.
- Stage.gd가 템플릿별로 분기 (카메라/goal/spawn 동작 다르게)
- 작업량: 중간 (2~3 세션)

**C안 — 부분 변환 (가장 보수)**
- 현재 가로 박스를 유지하되, 한두 맵만 형태 바꿈 (외벽 옥상=수직, 핵심부=아레나)
- 나머지는 그대로
- 작업량: 작음 (1 세션)

→ 추천: **B안**. A는 과하고 C는 user의 단조로움 불만 다 못 푼다.

### 3.2 특수 방 도입 범위

후보 (3.2 구분):
1. **현재 11맵 중 일부를 특수 방으로 전환** (예: 핵심부 = 보스 방, 이스터에그는 hidden 추가 슬롯)
2. **신규 12~14번째 맵으로 추가** (route_history와 별개로 stage 분기에 등장)
3. **secret room 시스템** — 일반 맵 안에 숨은 트리거(잠긴 문 형식)로 진입

→ 추천: 1번 + 3번 조합. 핵심부를 보스 방으로 격상, 일반 맵에 숨은 트리거로 이스터에그 방 진입.

### 3.3 보스 방 메커닉

- 보스 적 1마리 — patrol/sniper/drone 베이스 행동에 페이즈 추가?
- 또는 완전 신규 적 타입 (큰 사이즈, 여러 약점, 텔레그래프)
- HP 두 자릿수 (현재 적은 1~3). 사망 시 결말 분기 영향?

### 3.4 도전 방 룰 (있다면)

- 어떤 챌린지? (예: 30초 안에 끝, 1뎀 받으면 실패, 사격 금지 회피 only)
- 클리어 보상 (영구 강화? 일회성 부스트?)
- 실패 페널티 (없음? trust -1? 다음 stage 적 +20%?)

### 3.5 진입 조건

- 특수 방을 항상 보이게 vs 숨김?
- 숨김이면 트리거 종류 (특정 적 처치 / 특정 루트 조합 / 시간 / 무손상 등)
- 발견 시 VEIL 반응 (호기심? 경고?)

### 3.6 부스 환경 영향

1회 8~15분 안에 들어가야 함:
- 특수 방 길이 짧게 (각 1~2분)
- 보스 방은 마지막 stage라 5분까지 OK
- 이스터에그 방은 30초~1분
- 너무 많이 추가하면 8분 컷 못 맞춤
