## 각 맵 layout 명세

### 공통 기준
- 캔버스: 4400×720px (기존 유지)
- 플레이어 점프 상승: ~104px (단독), 이중점프 시 추가 ~80px
- 대시: 130px 수평
- 분기 진입점: x=600~800 (시작 후 약 1~2 화면)
- 분기 합류점: x=3400~3600 (골 전 약 1화면)
- GROUND_Y=600 기준

### 1. 외곽 진입로 (back_alley)
**컨셉**: 튜토리얼급 단순 직선. 처음 접하는 맵. 분기 없음.
**구조**: 단일 경로. 낮은 platform 3~4개로 계단식 이동.
**수직성**: 낮음

```
layout:
  platforms:
    - [x=400,  y=520, w=160]   # 낮은 계단
    - [x=700,  y=460, w=160]
    - [x=1100, y=520, w=200]
    - [x=1500, y=460, w=160]
    - [x=1900, y=520, w=200]
    - [x=2300, y=460, w=160]
    - [x=2700, y=520, w=200]
    - [x=3100, y=460, w=160]
    - [x=3500, y=520, w=160]

분기: 없음 (직선)
일방통행: 해당 없음

적 배치 가이드:
  - patrol x=900,1600,2400  y=ground  (3마리 -- stage 0이라 소수)
  - 저격수 없음 (stage 0 전용)
  - 드론 없음

보상: 기본 (분기 없으므로 보상 차등 없음)
```

### 2. 외벽 옥상 (rooftops)
**컨셉**: 수직 상승. 위로 올라갈수록 위험하고 보상이 크다. 떨어지면 아래 층으로 내려가며 돌아오지 못한다.
**구조**: 3층 수직 구조. 시작은 1층(ground), 진행하면서 위로 오를지 아래에 머물지 선택.

```
  3층 (y=220): 위험. 저격수 2마리 + 노출 구간. XP orb +2개.

layout:
  # 1층 platform (ground 바로 위 낮은 발판)
  - [x=500,  y=540, w=200]
  - [x=900,  y=540, w=200]
  - [x=1400, y=540, w=200]
  - [x=2000, y=540, w=200]
  - [x=2600, y=540, w=200]
  - [x=3200, y=540, w=200]

  # 2층 platform
  - [x=600,  y=380, w=200]
  - [x=1100, y=380, w=180]
  - [x=1600, y=380, w=200]
  - [x=2200, y=380, w=200]
  - [x=2800, y=380, w=200]
  - [x=3300, y=380, w=200]

  # 3층 platform (점프 2회 또는 이중점프 필요)
  - [x=800,  y=220, w=160]
  - [x=1300, y=220, w=140]
  - [x=1800, y=220, w=160]
  - [x=2400, y=220, w=140]
  - [x=3000, y=220, w=160]

  # 층 간 이동 사다리/발판
  - [x=720,  y=300, w=60]   # 2층→3층 중간 발판
  - [x=1220, y=300, w=60]
  - [x=1920, y=300, w=60]

일방통행:
  - 3층 → 2층: drop_through 가능
  - 2층 → 3층: 점프 가능 (이중점프 권장)

  drone:   천장 근처 자유 이동 (y=100~160 spawn, 2층 이상 진입 시 어그로)

보상:
  3층 도달 시: XP orb +2개 추가 spawn (x=1800, y=200 부근)
  1층만 통과 시: 기본 보상
```

### 3. 지하 인입로 (sewers)
**컨셉**: 깊이 하강. 상단은 좁고 빠른 길, 하단은 넓지만 함정이 많다. 분기 후 다시 올라와 합류.
**구조**: 시작은 중층. 상단 우회로(좁고 빠름) / 하단 주 통로(함정 + 보상) 2분기.

```
분기 구조:
  시작(x=140, y=600) → 분기점(x=700)
    상단(y=420): 좁은 파이프 통로. 함정 없음. 적 소수. 빠른 통과.
    하단(y=600): 넓은 하수도. 가시 함정 3개. 적 많음. XP orb +2개.
  합류(x=3400, y=520)

layout:
  # 상단 루트 (y=420 기준)
  - [x=700,  y=420, w=300]
  - [x=1100, y=420, w=200]   # 좁은 구간 시작
  - [x=1400, y=380, w=120]   # 더 좁음 (천장 낮음 표현)
  - [x=1600, y=420, w=200]
  - [x=1900, y=400, w=120]
  - [x=2100, y=420, w=300]
  - [x=2500, y=420, w=200]
  - [x=2800, y=440, w=300]
  - [x=3200, y=420, w=200]

  # 하단 루트 (ground=600 기준, 중간 platform 없이 지면 이동)
  # 함정(가시) 위치: x=1200,1800,2400 (y=600 지면 위)

  # 분기 진입 발판
  - [x=600,  y=460, w=80]    # 상단으로 올라가는 발판
  - [x=650,  y=420, w=80]

  # 합류 발판
  - [x=3300, y=460, w=160]
  - [x=3400, y=520, w=200]

일방통행:
  - 상단 → 하단: drop 가능 (하단 진입 후 상단 복귀 불가 — x=1200 이후 상단 발판 없음)
  - 하단 → 상단: x=3000 이전 불가, 합류 지점에서만 올라옴

적 배치 가이드:
  patrol: 상단 x=1000,1700 / 하단 x=1400,2000,2600
  bomber: 하단 x=1600,2200  (좁은 하수도에서 자폭병이 압박 극대화)
  drone:  없음 (지하라 드론 미등장)

보상:
  하단 클리어 시: XP orb +2개 (x=2800 지면)
  상단 통과 시: HP 회복 1개 (x=2000, y=400)
```

### 4. 폐쇄 지하철 (subway)
**컨셉**: 수평 2층 구조. 열차 위(지붕) vs 열차 아래(지면). 천장이 낮아 이동 제약.
**구조**: 지면 통로 + 열차 지붕 위 통로. 양방향 이동 가능하지만 천장이 낮아 점프 높이 제약.

```
분기 구조:
  지면(y=600): 표준 이동. 방패병이 좁은 통로를 막음.
  열차 지붕(y=380): 위 통로. 이중점프 없으면 접근 어려움. 적 소수. XP orb.

  열차 객차 표현: x=800~1400, x=1800~2400, x=2800~3200 구간에 지붕 platform 배치
  객차 사이 간격(x=1400~1800 등)은 지붕→지면 낙하 가능 구간

layout:
  # 열차 지붕 (y=380)
  - [x=800,  y=380, w=600]   # 1번 객차 지붕
  - [x=1800, y=380, w=600]   # 2번 객차 지붕
  - [x=2800, y=380, w=400]   # 3번 객차 지붕

  # 지붕 올라가는 발판 (객차 측면)
  - [x=750,  y=480, w=60]
  - [x=780,  y=420, w=60]
  - [x=1750, y=480, w=60]
  - [x=1780, y=420, w=60]

  # 지면 platform (낮은 장애물)
  - [x=500,  y=560, w=120]
  - [x=1500, y=540, w=100]   # 객차 사이 잔해
  - [x=2500, y=560, w=100]
  - [x=3300, y=560, w=120]

일방통행: 없음 (양방향). 단, 열차 지붕에서 낙하는 자유, 지붕 복귀는 점프 필요.
천장 높이: y=280에 천장 표시 (시각적으로만 — 실제 콜리전은 구현 여부 선택)

적 배치 가이드:
  patrol:  지면 x=1000,2000,3000
  shield:  지면 x=1600,2600  (좁은 통로에서 정면 블록 — 대시 또는 위 우회 강제)
  sniper:  열차 지붕 x=1200,2000  (y=380 위에서 아래 조준)
  drone:   없음 (천장 낮아 드론 미등장)

보상:
  지붕 루트: XP orb +2개 (x=2000, y=360)
  지면 루트: HP 회복 없음 (방패병 처치 보상으로 충분)
```

```
  합류(x=3400, y=200)   -- 상단에서 합류

layout:
  # 우측 계단 루트 (안전, y 감소하며 상승)

  - [x=3300, y=200, w=200]

  # 좌측 파이프 루트 (빠른 수직, 좁은 발판)

  - [x=680,  y=400, w=80]
  - [x=760,  y=320, w=80]

  # 분기 발판
  - [x=600,  y=520, w=100]   # 분기 전 공통 발판

일방통행:
  - 파이프 루트: 위→아래 drop 가능하지만 아래 파이프에서 위 복귀 불가 (좁아서 상징적으로)

  patrol:  계단 x=1200,1800,2400  (각 계단 층)

  sniper:  계단 x=2000  (y=220에서 아래 조준)
  bomber:  없음

보상:
  파이프 루트(좌측): XP orb +2개 (x=900, y=160)
  계단 루트(우측): 상단 합류 전 HP 회복 (x=3100, y=200)
```

## 적 배치 공통 규칙 (Stage.gd._spawn_enemies 재작성 가이드)

현재 RNG 흩기 방식에서 **지형 좌표 인지 방식**으로 전환.

```gdscript
# 각 맵의 spawn_guide 딕셔너리 구조
{
  "patrol": [
    {"x": 800, "y": GROUND_Y, "layer": "ground"},
    {"x": 1600, "y": 400, "layer": "mid"},
  ],
  "sniper": [
    {"x": 1200, "y": 160, "layer": "high",
     "facing": "left"}   # 플레이어 진행 방향 반대로 조준
  ],
  "drone": [
    {"x": 1500, "y": 120, "layer": "ceiling",
     "patrol_range": 200}
  ],
  "bomber": [
    {"x": 2200, "y": GROUND_Y, "layer": "ground"}
  ],
  "shield": [
    {"x": 1600, "y": GROUND_Y, "layer": "ground",
     "facing": "left"}   # 항상 플레이어를 향해 정면
  ]
}
```

**저격수 배치 원칙**:
- 항상 platform 위 (layer="high" 또는 "mid")
- x 좌표는 플레이어가 해당 위치를 지나기 전 보이는 위치 (플레이어 진행 방향 앞쪽)
- 플레이어와 다른 layer에 있어야 "저격당하는" 감각 확보

**드론 배치 원칙**:
- y=80~160 (천장 근처) spawn
- patrol_range로 좌우 순찰
- 플레이어가 같은 layer(상층)에 올라와야 근접 공격 가능 → 올라갈 발판 필요

**자폭병 배치 원칙**:
- 좁은 통로 또는 함정 근처
- 회피 공간이 좁은 곳에만

**방패병 배치 원칙**:
- 분기 합류 지점 또는 좁은 평지
- facing: "left"로 고정 (항상 플레이어를 향함)
- 측면 접근 가능한 공간이 있어야 우회 의미 생김
