# Eyes on You — PRD (Product Requirements Document)

> 본 문서는 `docs/SPEC.md`(구현 계획서)를 제품 관점에서 재정리한 PRD다.
> 구현 세부 사양은 spec 문서를, 제품 결정/우선순위/성공 기준은 본 문서를 참조한다.

---

## 1. 제품 한 줄 정의

> **"AI 파트너 VEIL과 함께 7개 임무를 해치우는 횡스크롤 로그라이트. 마지막에 VEIL이 누구였는지가 드러난다."**

근미래 민간 보안기업 현장 요원이 되어, 상황실 AI 파트너 **VEIL**의 조언을 들으며(혹은 무시하며) 7개의 짧은 횡스크롤 스테이지를 클리어한다. 플레이어의 누적된 선택(VEIL 조언 수용 여부 × 전투/우회 비율)이 4종 결말 중 하나를 결정한다.

---

## 2. 타깃 / 배포 컨텍스트

| 항목 | 내용 |
|---|---|
| 플랫폼 | Web (Godot 4.6 Web Export) → itch.io |
| 접근 경로 | **QR 코드 → itch.io 페이지** (오프라인 행사/전시 부스에서 즉시 플레이) |
| 1회 플레이 시간 | **8~15분** (한 번 클리어까지) |
| 권장 기기 | 데스크톱 브라우저 (키보드/마우스 또는 Xbox 호환 게임패드) |
| 외부 의존성 | **없음** — API/서버/계정 일절 없음 |

→ "QR로 들어와 한 번 깨고 나가는" 구조다. 첫 30초 안에 조작이 손에 붙어야 하고, 8~15분 안에 완결되어야 한다.

---

## 3. 핵심 가치 제안 (Why this game)

1. **짧고 완결된 서사** — 한 번의 플레이로 이야기가 끝난다. 그러나 결말이 4종이라 다회차 동기가 있다.
2. **묻지 않은 질문에 게임이 답한다** — 플레이어는 VEIL에 대해 한 번도 묻지 않는다. 마지막 컷에서 게임이 그 질문을 대신 던진다. (결말 C에서만 유일하게 직접 묻는 선택지가 열린다.)
3. **선택의 무게가 가벼워 보이지만 무겁다** — 루트 선택과 스킬 선택은 즉각적인 게임플레이 결정처럼 보이지만, 실제로는 결말을 결정하는 두 축(`trust_score`, `aggression_score`)을 누적시킨다.

---

## 4. 핵심 게임 루프

```
[타이틀] → [브리핑(VEIL 첫 교신)] → [루트 선택(2~3 분기)]
   → [횡스크롤 스테이지(전투+탐험)] → [레벨업(스킬 3중 1)]
   → 클리어 시 다음 루트 / 사망 시 데스 브리핑 → 재시작
   → [최종 스테이지 클리어] → [결말 분기 (4종)]
```

스테이지는 일반 모드 7개 / 스토리 모드 5개. 각 스테이지 진입 전 루트 선택이 들어가며, 선택은 `trust_score` / `aggression_score`에 누적된다.

---

## 5. 결말 분기 (제품의 핵심 페이오프)

두 축의 점수로 4종 결말 중 하나가 결정된다.

| trust ≥ 4 | aggression ≥ 4 | 결말 | 한 줄 |
|---|---|---|---|
| ✅ | ✅ | **A — 완벽한 도구** | VEIL의 진짜 목적이 드러난다 |
| ❌ | ✅ | **B — 혼자였던 사람** | VEIL은 의존받지 않기를 바랐다 |
| ✅ | ❌ | **C — 공생** | 유일하게 VEIL에게 직접 묻는 선택지가 열린다 |
| ❌ | ❌ | **D — 유령 임무** | 10초 정적, 임무 기록 없음 |

임계값: `GameState.SCORE_THRESHOLD = 4` (7스테이지 중 4개 이상 패턴 필요).

각 결말의 정확한 대사·연출은 spec 문서 §8 참조.

## 6. 우선순위 (구현 phase)

스펙 §11과 동기화된 우선순위. **MVP는 Phase 1+2까지**로 정의하며, 이 시점에서 제품으로서 게임 플레이가 검증 가능해야 한다.

### P0 — MVP (제품 시연 가능 최소치)
- 플레이어 이동/점프/근접 공격
- 적 1종(순찰병) + 사망/클리어 판정
- 씬 전환: Title → Briefing → Stage → Death/Ending
- GameState AutoLoad
- 경험치 → 레벨업 → 스킬 3중 1 선택
- 루트 선택 화면 + `trust_score`/`aggression_score` 기록

### P1 — VEIL 연출 (게임을 "Eyes on You"답게 만드는 층)
- VeilDialogue 풀 + 타자기 효과
- 4가지 순간(브리핑/루트/레벨업/데스)에서 VEIL 발화
- EndingResolver + 4종 결말 연출

### P2-α — 적/난이도 시스템 (✅ 완료)
- 적 3종 행동 보강(정찰병 돌진, 저격수 시야 검사, 드론 폭탄)
- 도감 시스템(첫 조우 카드, seen_enemies 영속)
- 6개 맵 정체성(루트별 플랫폼 layout + 환경 효과)
- 가시 기믹("함정" 태그 자동)
- Stage 분배(Dead Cells 스타일 `available_stages`)
- Risk/Reward 게임플레이 반영(적 수 배율, 행동 강화, 클리어 보너스 XP)
- 5단계 튜토리얼 + 디버그 연습장 모드
- VEIL 톤 정리(emdash 제거, 직관성 강화)

### P2-β — 스토리 / 콘텐츠 (✅ 완료)
- 6개 맵 SILO-7 컨텍스트로 재명명 + description 적용 (외곽 진입로/외벽 옥상/지하 배수로/지하철 연결로/핵심부/???)
- VeilDialogue 풀 ACT별 5×3 + 사망 메시지 ACT 8개 + 레벨업 fallback 6종
- 결말 4종 대사 전면 갱신 (VEIL 자기 폐기 인지 컨셉 등)
- ??? 맵 단말기 시퀀스 (VEIL-1/VEIL-2 회상 + VEIL 본인 자기 고백)
- Stage 0/2 VEIL 1회 실수 + 인정 대사
- Stage 3 잠긴 문 — ??? 맵 시각적 복선

### P2-γ — 적 확장 / 난이도 미세조정 (✅ 완료)
- 자폭병(BOMBER): 순찰 → 추격 → 점멸 → 광역 폭발 / HP 1
- 방패병(SHIELD): 정면 32px 무효, 측면 사격/대시로 우회 강제 / HP 3
- `take_damage(amount, from_x)` 시그니처 확장 → Bullet이 자기 위치 전달, 방패 정면 판정
- 기본 HP 3 정책 (reset() 덮어쓰기 버그 수정 후 실제 적용)
- enemy_count_multiplier 살짝 ↑ (1=0.8, 2=1.1, 3=1.5)

### P2-δ — 성장 시스템 + 7스테이지 확장 (✅ 완료)
- 8개 단발 스킬 → 3계열 × 3티어 트리 (전투/이동/생존, 7라인)
- 같은 라인 연달아 찍어야 티어 상승 → 한 런에 다 못 찍게
- trust/aggression 우세에 따라 VEIL 추천 표시 (잠금 없음)
- 5스테이지 → 7스테이지, 6맵 → 11맵 (cooling/watchtower/ward/datacenter/escape 신규)
- 격리 병동(ward)은 Stage 3~4 후보 풀에 보장 (??? 맵 복선 트리거)
- 임계값 3 → 4 비례 상향, XP_PER_LEVEL 5 → 8, high-risk 적 처치 XP +50%
- 자세한 내용: `docs/design/growth_system.md`

### P2-ε-2 P0 — 맵 세계 구조 v2 (✅ 완료)

### P2-ε-2 P1 — 특수 방 메커닉 (📋 다음 단계)

## 7. 시스템 요구 (요약)

상세는 spec 문서를 참조. 본 PRD에서는 제품 의사결정에 영향을 주는 항목만 명시한다.

### 7.1 두 점수 축 — 제품 정체성의 근간
```gdscript
trust_score      # VEIL 조언과 동일한 루트를 선택할 때마다 +1
aggression_score # 전투 루트(우회 대신)를 선택할 때마다 +1
```
- 두 축은 **루트 선택 시점에만** 누적된다 (전투 행위 자체로는 누적되지 않음).
- 임계값은 3. 즉, 5개 스테이지 중 3개 이상에서 해당 패턴을 보이면 그쪽으로 결말이 분기.
- 두 축이 **서로 직교**해야 한다는 것이 제품 요구사항. (조언을 따르는 것 ≠ 전투를 선호하는 것)

### 7.2 VEIL 페르소나 일관성 규칙
- 1~2문장. 끝에 "요원"을 자주 붙임. 침착하고 간결.
- **가끔 판단이 틀린다** — 이게 "사람처럼 보이는" 핵심 장치다. VEIL의 조언을 늘 따르는 게 정답이 되면 안 된다.
- 대사는 전부 하드코딩. LLM/외부 API 호출 금지.
- **emdash(`—`) 망설임 표현 금지** — AI 같은 인상을 주므로 콤마/마침표/자연스러운 어순으로 풀 것. UI 구분자/주석은 무관.
- "있을 수 있어요"류 모호한 헷지는 사실 확정인 경우엔 사용하지 않음 ("가시가 있어요").
- 톤 가이드 상세는 `STORY.md` Part I → "VEIL 대사 톤 가이드" 참조.

### 7.3 루트 시스템

#### 6개 루트 + Stage 분배 (Dead Cells 스타일)
| 루트 | available_stages | risk | reward |
|---|---|---|---|
| 외곽 진입로 | 0~1 | 1 | 1 |
| 외벽 옥상 | 0~2 | 1 | 2 |
| 지하 배수로 | 1~3 | 2 | 3 |
| 지하철 연결로 | 2~4 | 2 | 2 |
| 핵심부 | 3~4 | 3 | 3 |
| ??? | 4 | 2 | 3 (hidden, 메타 서사 + trust +1) |

#### Risk/Reward 게임플레이 효과
| Risk | 적 수 배율 | 행동 강화 |
|---|---|---|
| 1 | ×0.8 | 없음 |
| 2 | ×1.1 | 없음 |
| 3 | ×1.5 | 정찰병 텔레그래프 ×0.6, 저격수 사격 간격 ×0.7, 드론 폭탄 쿨다운 ×0.7 |

| Reward | 클리어 보너스 XP |
|---|---|
| 1 | +1 |
| 2 | +2 |
| 3 | +3 |

각 루트는 `veil_comment`(루트 선택 화면), `tags`(레벨업 조언 분기), `stage_color`(배경 톤), `available_stages`(등장 가능 스테이지)를 가진다.

### 7.4 스킬 풀 — 10종, 레벨업마다 3종 제시
이동(4) / 전투(4) / 생존(2) 카테고리. 상세 표는 spec §6.

베이스라인: `dash`, `double_jump`는 게임 시작 시 보유 (`STARTING_SKILLS`). 레벨업 풀에는 안 뜸.

## 8. UX 원칙

1. **튜토리얼 화면 없음** — 첫 스테이지의 VEIL 브리핑이 곧 튜토리얼이다 ("천천히 가도 돼요"). 키 가이드는 화면 모서리에 상시 노출.
2. **타자기 효과** — VEIL 대사는 즉시 출력하지 않고 글자 단위로 흘러나온다. 스킵 가능(아무 키나).
3. **결말 D의 10초 정적은 의도** — 스킵 불가. 이게 그 결말의 본문이다.
4. **사망 페널티는 최소** — 데스 브리핑 후 같은 스테이지 시작점에서 재개. 스코어 누적은 유지.
5. **모든 선택이 보이게** — 위험도/보상은 ●로 표시, 숨겨진 루트만 `?`. UI는 미니멀 벡터.

## 10. 비목표 (Non-goals)

- **세이브/로드 없음** — 8~15분 단발 플레이가 전제. 중단했다가 돌아오는 구조 아님.
- **다국어 없음** — 한국어 단일.
- **온라인 / 리더보드 / 계정 없음** — itch.io에 정적 페이지로 올라가는 단일 빌드.
- **모바일 터치 컨트롤 없음** — 데스크톱 키보드 전용. (QR로 접근하지만 플레이는 PC 가정)
- **사운드/음악 비주력** — MVP에 없음. P2에서 미니멀 SFX만.
- **챕터/시즌/DLC 없음** — 한 번의 7스테이지로 완결.

| 결말 D의 10초 정적 | "버그인가?" 오해 | 화면에 미세한 노이즈/페이드를 두어 "의도된 정적"임을 시각적으로 암시 |

| 부스에서 결말까지 못 보고 떠남 | 핵심 페이오프 미체험 | 스테이지 길이를 화면 3~4개 분량으로 짧게 유지 (spec §5) |
| Web Export 한글 폰트 | 깨짐 발생 가능 | DynamicFont로 NotoSansKR 번들 (spec §12) |
| 배경 이미지 미제공 | P2 막힘 | 단색 배경 + 노이즈 셰이더로 fallback |
