# 성장 시스템 확장 제안서

> 다음 작업자(Claude)에게 넘기는 설계 브리프. 구현 전 사용자와 한 번 합의 후 들어갈 것.

## 배경 — 현재 상태

- **스테이지 수**: `GameState.TOTAL_STAGES = 5`. 각 스테이지에서 루트 1개 선택.
- **스킬 풀**: `SkillSystem.ALL_SKILLS` 8종. 베이스라인 2종(`dash`, `double_jump`)은 자동 보유.
- **레벨업 흐름**: 적 처치 → ExpOrb → `XP_PER_LEVEL = 5` 누적 → 레벨업 시 3개 중 1개 선택. **한 스킬은 1회만 획득** (`roll_choices`가 owned에서 제외).
- **레벨/스킬 보존**: 튜토리얼 → 본편은 스킬만 유지, 레벨/XP는 리셋 (`GameState.start_main_game`).

### 핵심 문제

1. **5스테이지 + 1픽/레벨 + 8스킬 풀** → 후반부엔 거의 모든 스킬을 소거법으로 찍게 됨 → 빌드 분화가 사실상 없음.
2. 스킬 하나당 단일 효과로 끝나서 **"성장하는 느낌"이 약함**. 한 번 찍고 끝.
3. 루트(risk/reward)나 신뢰/공격성 스코어가 빌드 선택과 별로 엮이지 않음 — 스토리 갈래는 있지만 메커닉이 따라가지 못함.

## 목표 (사용자 요구)

1. **레벨 수 ↑** — 성장 시간 늘리기.
2. **특성 다양화** — 빌드가 갈리는 선택지.
3. **티어/누적 강화** — 같은 특성을 여러 번 찍으면 단계적으로 강화되는 느낌.
4. **자유도** — 한 런에서 모든 걸 다 못 찍게. 포기/특화의 무게.
5. **스토리/밸런스와 결합** — 단순 수치 인플레가 아니라 루트·VEIL 신뢰도와 엮인 시스템.

## 제안 방향

### A안 — 티어형 스킬 트리 (추천)

기존 8개 스킬을 **3개 계열(전투/이동/생존)** × 각 **3티어**로 재편.

```
전투 계열
  T1 사격 강화 (+1 dmg)        → T2 (+2 dmg, 사격 시 잠깐 가속)   → T3 (관통 1체 추가)
  T1 삼연사 (3발 부채꼴)        → T2 (5발)                       → T3 (5발 + 약 추적)
  T1 폭발물 (3s CD)            → T2 (반경 +30%, CD 2.5s)         → T3 (2회 충전)

이동 계열
  T1 글라이드 (천천히 낙하)      → T2 (낙하 중 가속)                → T3 (글라이드 중 사격 패널티 제거)
  T1 대시 강화 (CD -20%)        → T2 (대시 거리 +30%)              → T3 (대시 후 0.3s 데미지 면역)

생존 계열
  T1 최대 HP +1                → T2 (+2, 피격 후 1s 무적)         → T3 (피격 시 짧은 슬로모)
  T1 비상 방어막 (1회 부활)      → T2 (회복량 1→2)                 → T3 (재충전형: 30s 후 재무장)
```

규칙:
- 같은 계열을 연달아 찍을 때만 티어가 올라감.
- T2/T3는 T1을 가진 사람한테만 후보로 등장.
- 한 런에 트리 하나를 T3까지 다 채우려면 레벨 9~10 필요 → 다 못 찍게 함.

**스토리 결합**:
- 신뢰도(trust) 높음 → VEIL이 **이동/생존 계열을 추천 표시**로 가중 (안전한 판단).
- 공격성(aggression) 높음 → **전투 계열 가중**.
- 이게 "VEIL이 너를 어떤 요원으로 보고 있는가"의 매커닉적 표현이 됨.

**밸런스**:
- `XP_PER_LEVEL`을 5→8 정도로 올리고, 적 처치 XP는 그대로 → 스테이지당 평균 1~2 레벨업.
- 5스테이지 × 1.5렙 = 평균 7~8 레벨업 → T1만 다 찍거나, 1계열을 T3까지 파거나의 양자택일.
- 루트 risk별 XP 보너스에 차등 (high-risk 루트는 처치 XP +50%) → 빌드 선택과 루트 선택이 서로 압박.

### B안 — 특성 포인트 (Hades 스타일)

레벨업 시 1포인트를 받아 자유 분배. 트리는 같지만 "3개 중 선택" 대신 "내가 원하는 데에 투자". 자유도 ↑↑, 결정 피로 ↑.

→ A안이 결정 부담 적고 페이즈가 명확해서 우선 권장.

### C안 — 부분 도입 (최소 변경)

기존 8스킬을 그대로 두되 **각 스킬에 +1, +2 강화 단계만 추가**. 트리 재설계 없이 누적 픽만 허용. 가장 변경량 적지만 "다양화" 욕구는 못 채움.

## 구현 시 손볼 곳 (참고)

- `scripts/SkillSystem.gd:11` — `ALL_SKILLS` 구조 변경. 티어/계열 필드 추가.
- `scripts/SkillSystem.gd:22` — `roll_choices` 로직 재작성 (owned 단순 제외 → 티어 prereq + 계열 가중).
- `scripts/GameState.gd:20` — `skills` 배열을 `Dictionary[String, int]` (id→tier)로 바꾸거나, id에 tier 접미사를 붙이는 방식 결정 필요.
- `scripts/GameState.gd:120-129` — `has_skill`, `add_skill`을 티어 인지하도록 수정.
- `scripts/Player.gd` — 각 스킬 효과가 직접 코드로 박혀 있음. 티어값을 읽어 분기하도록 일반화.
- `scripts/LevelUpOverlay.gd` — UI에서 티어 표시 (T1/T2/T3 뱃지).
- `scripts/BestiaryData.gd` 옆에 `SkillTreeData.gd`를 새로 두는 것도 고려 — 데이터/로직 분리.
- `scripts/Stage.gd` — XP 보너스를 루트 risk와 묶는 부분 (현재 없음, 추가 필요).

## 사용자와 합의해야 할 것

구현 시작 전에 사용자한테 답을 받고 가야 할 결정들:

1. **A/B/C 중 어느 안인가?** (저자 추천 A)
2. **레벨 캡과 XP 곡선** — 평균 7~8레벨로 잡을지, 더 길게 가져갈지. 스테이지 수 자체를 5→7로 늘릴지도 같이 결정.
3. **계열 분류** — 위에 제시한 전투/이동/생존 3분류로 갈지, 다른 축(예: 정확/은폐/기동)을 쓸지.
4. **신뢰/공격성과의 결합 강도** — 약한 가중치(추천 표시만)인지, 강한 게이팅(특정 트리 잠금)인지. 강하면 스토리 임팩트 ↑, 자유도 ↓.
5. **튜토리얼 처리** — 현재 튜토리얼에서 스킬 1개 픽 후 본편 이전. 트리 시스템에선 그 1픽이 항상 T1이 되어야 일관됨. 튜토리얼 스킬 풀을 따로 둘지 결정.
6. **세이브 호환성** — `GameState.save_settings`에 스킬 저장이 안 들어 있는 듯하니 영향 작음. 도감(`seen_enemies`)은 별개로 유지.

## 비목표 (이번 작업에서 안 하기로 정한 것)

- 새 스킬을 8개 이상 추가하는 것 — 우선 기존 풀 재구성으로 시작.
- 무기 시스템, 장비 슬롯 같은 추가 축 — 스킬 트리만으로 충분히 분화 가능한지 먼저 확인.
- 메타 진보(런 간 영구 강화) — 별도 제안.
