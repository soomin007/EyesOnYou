## A. 플레이테스트로만 확인 가능 (내가 못 함 — 사용자 플레이 필요)

- **피드백 설문 응답 수집** → 아래 튜닝 항목의 외부 신호로 활용. 초보자 난이도·UX가 1차 목적. ([[feedback-form]])
- **내보낸 빌드 체감**: 해상도 실제 적용·텍스트 선명도·비네트 농도(0.10)·맵 간 어두움 지속. export 설정 도움 가능.
- **VEIL 추천 임계값 튜닝**: `PERF_SKILLED_HITS_MAX=1` / `PERF_STRUGGLE_HITS_MIN=4`(최근 2스테이지 평균)가 체감과 맞는지.
- **신뢰 단계 튜닝**: trust 임계(0/4/8/12), 수용률 0.5, 게이지 스케일[2,4,8,12,16], 취약함 게이트 체감(무사망→COLD가 너무 차갑진 않은지).
- **스킬 외형 점검**: 전투 중 총기 회전·부착물 겹침·가독성, 획득/티어업 플래시 체감, 활강 유도↔사격강화 관통 분리 체감.
- **함정/저격수 난이도**: 후반 트랩(datacenter·watchtower), 둥지 저격수 LoS·사선 노출, 글라이드 게이트 도달 난이도.
- **shield T3 재충전**: 재충전 시간(30s)·재무장 플래시 톤.
- **포스터 2배 고해상 인쇄 가독성** + 필요 시 PNG→PDF 외부 변환.

## B. 콘텐츠 / 게임플레이 (방향 결정 필요)

- **글라이드 유리한 맵**: 저격병을 글라이드로만 닿거나 회피하는 위치(고지대 단독 발판 / 긴 갭 너머 / 긴 낙하 구간)에 둔 맵.
  신규 맵 vs 기존(rooftops/watchtower) 개조 — 방향 미정.
  - `map_audit.md` 결과: 빈칸은 "유리 지형 부재"가 아니라 ① **회피 강제 저격수**(못 잡고 지나가야 하는 위치)
    ② **유인 강화**(글라이드/대시로만 닿는 지름길·보상).
- **유도 차지 사격 스킬**(신규 아이디어): 공격을 꾹 누르면 연사 대신 누른 시간만큼 유도되는 강탄. 매력적이나
  차지 시스템 신규 + multishot T3 / glide T3 유도와 역할 중복 검토 필요.
- **공격 속도 라인 부재**: 전용 공속 패시브 없음(fire_boost T2의 짧은 가속뿐). 위 차지 스킬로 대체할지 검토.

## C. 코드 정리 (착수 시 플레이테스트 동반)

- **Stage.gd 분리** — 3580줄 god-object, 수십 개 공유 멤버로 엮임. 자율 보류. 단계적 안전 계획:
  ① 가장 독립적인 `_ambience_*`(12개, 순수 장식)를 `StageAmbience` 헬퍼로 추출 → ② HUD 빌드/갱신 묶음
  → ③ 적 스폰/웨이브. 각 단계 후 전 스테이지 타입 플레이 확인. 사용자가 함께 검증할 때 착수 권장.
- **Pretendard-Medium 도입**: 외부 폰트 자산 필요. 전역 아웃라인 테마로 "중간 굵기" 체감 확보돼 우선순위 낮음.
