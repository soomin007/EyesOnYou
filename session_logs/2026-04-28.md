## 결정 사항
- **PRD vs Spec 분리**: 제품 의사결정은 PRD가 우선, 구현 디테일은 spec이 우선이라 명시 — 두 문서가 충돌할 때의 단일 결정 기준 확보
- **씬 절차적 빌드**: `.tscn`은 노드 트리 + 스크립트만 담고, 모든 게임 객체(플랫폼/적/플레이어/HUD)는 `Stage.gd._ready()`에서 코드로 생성. 이유: 루트 태그에 따라 적 구성과 배경색이 바뀌는 동적 스테이지를 만들기 쉬움
- **레벨업은 별도 씬이 아닌 오버레이**: 스테이지 진행 중 XP가 차면 `CanvasLayer` 오버레이로 표시. 별도 씬으로 빼면 스테이지 상태(적 위치, 카메라, 진행도)가 사라짐
- **자식 노드 추가 순서**: `add_child(parent)` → `parent.add_child(visual)` 순서로 하면 `@onready`가 visual을 못 찾음. 자식부터 모두 부착 후 트리에 추가하는 패턴으로 통일
- **`Array[T]` 회피**: Dictionary 안의 배열 리터럴은 항상 untyped Array라서 `Array[String]` 변수에 대입 시 런타임 에러. 일관되게 `Array`로 선언하기로 결정 (요소별 append로 typed 유지하는 방법도 있지만 코드량이 늘어남)

## 디버깅 노트
- `current_route_tags = route.get("tags", []) as Array[String]` → `Trying to assign an array of type "Array" to a variable of type "Array[String]"`. 원인: Dictionary 내부 배열 리터럴은 요소 타입이 없는 `Array`. `as Array[String]` 캐스팅도 런타임에 실패. 같은 패턴이 8개 위치에 퍼져 있어 일괄 수정 (GameState/EndingResolver/Ending/RouteData/SkillSystem/RouteMap/Stage)
- `briefing.tscn`의 PanelContainer가 자식 VBoxContainer를 그대로 감싸 텍스트가 패널 모서리에 붙음. PanelContainer 자체는 padding 속성이 없어 `MarginContainer`를 중간에 끼워 해결. `@onready` 경로(`$Box/V/...` → `$Box/Margin/V/...`)도 함께 갱신 필요
