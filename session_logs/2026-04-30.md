## 결정 사항
- **셰이더로 흰배경 제거**: 외부 PNG 자산이 투명 배경을 안 가져도 작동하게. 하드코딩된 임계값 0.95는 거의 흰색만 자르고 본문 색은 보존
- **스프라이트 fallback to PlaceholderTexture2D**: `assets/sprites/*.png`가 아직 없는 상태에서도 게임이 돌도록. 자산이 들어오면 자동으로 교체
- **Player/Enemy의 자체 비주얼 빌드**: 기존 Stage가 ColorRect를 자식으로 박아주던 패턴 폐기. `set_script` 후 `_ready`에서 `_setup_visual()` 호출. CollisionShape2D는 여전히 Stage에서 생성(콜리전 크기는 스프라이트 스케일과 분리)
- **Tutorial은 별도 씬, 한 번만 실행**: `GameState.tutorial_done`은 `reset()`에서 보존되며 `settings.cfg`에 영속화. Web Export에서는 브라우저 localStorage에 저장
- **Tutorial에서 dash 스킬 임시 부여**: 스킬 시스템을 우회하지 않고 `GameState.skills.append("dash")` 후 `_finish_tutorial`에서 `reset()`으로 정리
- **가시 구간 게이팅은 데미지 대신 배리어**: 스파이크는 시각 장식 + Area2D 검출용. `dash_timer > 0` && in_zone 시 배리어 제거. 데미지로 처벌하면 사망→Death 씬 진입으로 튜토리얼 흐름이 깨짐
- **PauseHelper는 RefCounted 정적 헬퍼**: Stage와 Tutorial 두 군데서 동일 오버레이 사용. 콜백은 `Callable`로 받아 결합도 낮춤
- **Settings는 단일 씬, 양쪽 컨텍스트에서 재사용**: Title/Pause 양쪽에서 `instantiate()`로 자식 추가. 둘 다 `closed` 시그널로 닫음
- **`pause` 액션을 키바인딩 변경 대상에 포함**: 사용자가 ESC 외 다른 키를 원할 수 있음. 단 캡처 중 ESC를 누르면(설정 열기 자체를 ESC로 두려는 경우 외에는) 캡처 취소

## 디버깅 노트
- `dummy.killed.connect(_on_dummy_killed)`은 `dummy: Node2D` 타입에서 정적 분석상 위험 → `dummy.connect("killed", _on_dummy_killed)` 문자열 기반으로 변경. (Stage.gd의 기존 `e.killed.connect`는 동일 패턴이지만 그대로 둠 — 런타임에 동작)
- `player.dash_timer > 0.0` 직접 접근은 `player: CharacterBody2D` 타입이므로 정적 분석 실패. `float(player.get("dash_timer"))`로 우회
- `project.godot` `ui_skip` 두 번째 InputEventKey의 `unicode`가 0이 아닌 32 — 기존 파일과 정확히 매칭하려면 raw 값을 그대로 봐야 함
