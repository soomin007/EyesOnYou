# 2026-05-01 세션 로그

## 세션 1 — 요약
PNG 스프라이트 + 흰배경 셰이더 접근을 폐기하고, 콜리전 박스에 정확히 맞는 단순 벡터 캐릭터로 교체. 튜토리얼은 6단계로 확장(레벨업/스킬 단계 추가). 맵 디자인을 Stage·Tutorial 양쪽 다 다시 잡고, 단일점프로 도달 가능한 계단형 플랫폼 레이아웃을 5종 마련.

## 주요 변경
- `scripts/CharacterArt.gd` 신규 — `Polygon2D` 합성으로 Player/Patrol/Sniper/Drone/TutorialDummy 빌드. 모든 비주얼이 콜리전 박스(28×56 / 28×40 / 32×24) 안쪽에서만 그려짐
- `scripts/LevelUpOverlay.gd` 신규 — 레벨업 오버레이 빌더. Stage/Tutorial 공용
- `scripts/Player.gd` — Sprite2D/ShaderMaterial/PlaceholderTexture2D 코드 제거. `CharacterArt.build_player()` 호출. 좌우 반전은 `visual.scale.x = ±1`. 사격 트레이서/머즐 플래시는 그대로 유지
- `scripts/Enemy.gd` — `_setup_visual` 폐기. enemy_type에 따라 `CharacterArt.build_*`. flip은 `visual.scale.x`
- `scripts/TutorialDummy.gd` — `CharacterArt.build_tutorial_dummy()` 사용. signal에 position 전달하도록 시그니처 통일
- `scripts/Stage.gd` — 맵 디자인 개선: 천장 그라디언트, 멀리 있는 실루엣 기둥(parallax 느낌), 빛기둥, 플랫폼 윗면 라이트, 골 빛기둥. 5종 레이아웃 패턴(stage_idx % 5)으로 스테이지마다 다른 지형. 모든 플랫폼은 단일점프(상승 ~104px)로 도달 가능하도록 검증. `_show_levelup`은 `LevelUpOverlay.show()` 호출로 단순화
- `scripts/Tutorial.gd` 전면 재작성 — 6단계(이동/점프/공격/레벨업/대시/완료). 점프 구간은 3단 계단(P1 단일점프, P2 이중점프 필수, P3 P2에서 단일점프). 레벨업 더미는 LEVELUP 단계 진입 시점에서야 lazy 스폰(이전 단계에서 미리 처치 방지). XP는 사전 충전(`XP_PER_LEVEL - 2`)해서 2킬에 자동 레벨업 트리거
- `assets/sprites/`, `assets/shaders/` 폴더 전체 삭제 — 셰이더/PNG 접근법 폐기

## 결정 사항
- **벡터 합성 vs PNG**: PNG는 (1) 콜리전 박스와 시각이 어긋남 (2) 셰이더로 흰 배경을 잘라도 외곽선이 거칠고 (3) 애니메이션도 없어서 정적인 그래픽으로는 단순 벡터가 더 깨끗함. PRD §9 "코드 생성 미니멀 벡터" 방침과도 일치
- **CharacterArt를 정적 헬퍼로**: `RefCounted` + static 메서드. 노드 트리에 합쳐 넣는 방식이 클래스 인스턴스보다 가벼움. 각 캐릭터의 폴리곤 좌표가 콜리전 박스 안에 들어가도록 직접 검증
- **레벨업 오버레이 추출**: Stage가 갖고 있던 inline 오버레이를 `LevelUpOverlay.show()`로 빼서 Tutorial이 그대로 재사용. 두 곳에서 같은 UI/레이아웃 보장
- **Tutorial XP 사전 충전**: 정상 5XP/레벨 룰을 굽히지 않으면서 튜토리얼에서 1~2킬로 레벨업이 발생하도록 `XP_PER_LEVEL - 2`만큼 미리 채움. 자연스러운 시퀀스: 적 처치 → 오브 자동 흡수 → XP 바 채워짐 → 레벨업 오버레이
- **레벨업 더미 lazy 스폰**: Tutorial._ready에서 미리 만들어두면 player 사격 사거리에 들어왔을 때 보이지 않는 상태에서 처치되는 사고 발생. `_advance_to(LEVELUP)` 시점에서야 인스턴스 생성하도록 변경
- **Stage 플랫폼은 단일점프 도달 가능**: 플레이어가 시작 시점에 어떤 스킬을 가질지 알 수 없음(이중점프는 선택사항). 모든 레이아웃은 단일점프 계단으로 풀 클리어 가능. 이중점프가 있으면 더 빠른 루트가 열리는 정도
- **PRESET_FULL_RECT의 함정 회피**: PauseHelper/LevelUpOverlay 모두 `CenterContainer + PRESET_FULL_RECT`로 정확히 화면 중앙 정렬. PRESET_CENTER 단독 사용은 글로벌 메모리에 적힌 좌상단 잘림 버그 때문에 회피

- `dummy.killed.connect(_on_attack_dummy_killed)` vs `dummy.connect("killed", ...)`: TutorialDummy의 `killed` 시그널이 `(at_position: Vector2)` 인수를 갖도록 통일했더니, 핸들러 함수도 `_on_xxx(_pos: Vector2)` 시그니처로 받아야 함. 이전엔 인수 없는 시그널이라 `_on_dummy_killed()` 였음 — 시그니처 불일치로 silent 호출 실패할 뻔함

### 결정 사항
- **공격 박스를 발끝까지 확장**: 트레이서는 가슴 높이(시각적 피드백) / 데미지 박스는 가슴~발끝(실제 판정). 두 개념 분리. 트레이서는 어디서 발사됐는지 보여주고, 데미지는 박스 내 모든 적에게 적용. CharacterArt 적은 28×40 콜리전이라 발이 ground level에 있음 — 데미지 박스가 발끝까지 안 닿으면 못 맞힘
- **One-way 플랫폼 표준화**: 모든 떠 있는 플랫폼은 `one_way_collision`. 지면/벽은 양면 그대로. 점프 중 머리 박는 부정적 경험 제거. 슈퍼마리오/Celeste 표준
- **저격수 조준 텔레그래핑**: 발사 0.7s 전 빨간 라인 노출. 플레이어가 회피/엄폐 가능. "보이지 않는 데미지"는 액션 게임 기본 룰 위반
- **튜토리얼 진입 점**: 첫 플레이 SPACE는 `start_after_title()` (자동 분기, 부스 환경에서 자연스러움). 명시적 "튜토리얼" 버튼은 `tutorial_done` 플래그를 무시 — 부스 운영자가 다음 손님 위해 다시 보여주거나, 개발 중 재테스트 가능
- **Player z_index = 2**: 적이 플레이어 위에 그려지는 가림 현상 방지. 플레이어는 항상 모든 게임 오브젝트 위. Sniper 트레이서는 z=2(같이), 조준 라인은 z=1

## 세션 3 — 스킬 시스템 정상화 + 진짜 총알

### 요약
"튜토리얼에서만 dash/double_jump 됨", "스킬 사용법 안 알려줌", "조작법에 스킬 키 없음", "총 들었는데 hitscan rect" 4건 동시 해결. 사격을 진짜 `Bullet` 객체로 교체하고, dash/double_jump를 베이스라인으로 승격, `skill` 액션(Q)을 신설해 explosive를 정상 동작하게 만듦.

### 주요 변경
- `scripts/Bullet.gd` 신규 — `Area2D` 기반 발사체. `body_entered`로 적/벽 감지. 적 처치 후 `queue_free`(piercing 보유 시 관통 유지). speed/lifetime은 ranged 스킬로 +50%
- `scripts/SkillSystem.gd` 재구성 — dash/double_jump 풀에서 제거(베이스라인). stealth/roll → piercing/multishot로 교체(둘 다 패시브, 사격 강화). 각 스킬에 `active`/`key` 메타데이터 추가
- `scripts/GameState.gd` — `STARTING_SKILLS = ["dash", "double_jump"]`, `start_main_game()` 추가(skills 보존하는 부분 reset). `KEYBIND_ACTIONS`에 "skill" 추가
- `scripts/Player.gd` — hitscan rect 폐기, `Bullet.new()` 발사. `_try_skill()` (Q): 폭발물 보유 시 반경 180 AOE 폭발(데미지 3, 3s 쿨다운, Polygon2D 원이 확장+페이드). `wall_slide` 구현(`is_on_wall_only` && 낙하 중 → vy 클램프 140)
- `scripts/TutorialDummy.gd` — `Node2D` → `StaticBody2D`(layer 4). Bullet의 `body_entered`가 감지하도록
- `scripts/Tutorial.gd` — dash/double_jump 임시 부여 코드 삭제(베이스라인이라 이미 보유). `_finish_tutorial`은 `start_main_game()` 호출(스킬 보존). 레벨업 후 `_update_levelup_sign(picked_id)`이 표지판을 "[스킬명 획득] / [Q] 키로 사용 또는 자동 적용" 로 갱신
- `scripts/Settings.gd` — 조작법 탭에 "액티브 스킬" 행 추가, 기본값 Q
- `scripts/Stage.gd` — HUD 키 안내에 "Q 스킬" 추가
- `scripts/VeilDialogue.gd` — 삭제된 stealth 참조를 wall_slide로 교체
- `project.godot` — `skill` 액션(KEY_Q, physical_keycode 81)

### 결정 사항
- **dash + double_jump을 베이스라인으로**: 튜토리얼이 가르친 능력이 본편에서 사라지는 inconsistency 해소. 이 둘은 "이동의 기본기"라 간주, 풀에는 빼고 항상 부여. SkillSystem 풀에서는 제외. 다른 스킬 8종이 레벨업 픽 풀
- **start_main_game() 분리**: 기존 reset()은 Title 진입 시 풀 리셋용. 튜토리얼 → 본편 전이는 진행 상태(stage/xp/score)만 초기화하고 skills는 보존. 두 함수가 공유하는 진행 필드는 명시적으로 양쪽에 적어 의도가 보임
- **공격을 진짜 Bullet으로**: rect-hitscan은 즉시 판정이라 사거리/속도/벽 차단 같은 게임플레이 변수가 안 붙음. Area2D 발사체로 바꾸니 piercing/ranged/multishot 같은 사격 변형 스킬이 자연스럽게 구성 가능. `body_entered`로 적·벽 모두 같은 코드 경로
- **TutorialDummy를 StaticBody2D로**: 이전엔 Node2D + global_position 검사로 hitscan에 맞췄음. Bullet이 PhysicsBody2D만 감지하므로 StaticBody2D(layer 4)로 승격. 이동 안 함은 동일
- **explosive를 active로 확정**: stealth/roll은 구현 비용 대비 효용 낮아서 풀에서 제거. piercing/multishot는 단순 패시브 + 사격 시각 차이가 명확. explosive만 키 입력 필요한 액티브로 남기고 `skill` 액션(Q) 한 슬롯만 운영
- **레벨업 픽 후 표지판 갱신**: 픽한 스킬의 `active`/`key` 메타데이터 읽고 동적으로 사용법 출력. `OS.get_keycode_string`으로 현재 바인딩된 키를 표시 → 사용자가 Settings에서 Q 외 다른 키로 바꿔도 표지판이 정확

- Bullet의 trail ColorRect 위치: 진행 방향 반대편(뒤쪽)에 그려야 함. dir>0(우진) → trail x=-20, dir<0(좌진) → trail x=0~20. 부호 한 번 잘못 두면 트레일이 총알 진행 방향 앞에 매달려 보임

## 세션 4 — 입력 체계 개편 (마우스 + WASD)

### 요약
TutorialDummy 스크립트 타입 에러(StaticBody2D ≠ Node2D) 해결. 사격/스킬을 마우스 좌/우 클릭에 기본 할당(J/Q는 보조). 점프 primary는 W. WASD를 ui_left/right/up/down에도 바인딩해서 메뉴/스킬 선택을 키보드 한 손으로 가능. settings.cfg 스키마 v2(키+마우스 혼합 지원).

### 주요 변경
- `scripts/TutorialDummy.gd` — `class_name TutorialDummy` 추가
- `scripts/Tutorial.gd` — `Node2D.new() + set_script(...)` 패턴을 `TutorialDummy.new()`로 교체. 필드 타입도 `Node2D` → `TutorialDummy`. `dummy.connect("killed", ...)` → `dummy.killed.connect(...)` 정적 시그널 접근으로 통일
- `scripts/Main.gd` — `_bind_default_mouse_inputs()`(마우스 좌→attack/우→skill, primary 슬롯 0에 삽입). `_bind_wasd_to_ui()`(W/S/A/D를 ui_up/down/left/right에 추가, idempotent). load_settings 호출 전에 실행
- `scripts/GameState.gd` — settings.cfg 스키마 v2로 승격. 입력 이벤트를 `{type: "key"|"mouse", code|button}` Dictionary로 저장. v1 cfg 로드 시 input 섹션 무시(스키마 호환 안 되어 키바인드만 폐기, audio/flags는 유지). 저장 시 `meta.version = 2`
- `scripts/Settings.gd` — `_event_label`이 InputEventKey/MouseButton 양쪽 처리. `_input` 캡처가 마우스 버튼도 받음(휠 제외). `_apply_default_keybindings`가 `[type, code]` 튜플 형식으로 마우스 default 포함. 힌트 문구 "키 또는 마우스 버튼" 으로 갱신
- `scripts/Stage.gd` HUD 키 안내 — "마우스 좌클릭 사격 / 마우스 우클릭 스킬"
- `scripts/Tutorial.gd` 표지판 — "W 키로 점프", "마우스 좌클릭으로 사격". `_label_for_action`이 마우스 버튼 라벨 "마우스 좌클릭/우클릭/가운데" 도 반환
- `project.godot` jump 이벤트 순서 변경: W primary, Space secondary

### 결정 사항
- **마우스 + 키 조합 default**: 사격이 가장 자주 쓰이는 액션이므로 마우스 좌클릭이 가장 직관적. 키는 보조로 J/Q 유지(왼손만으로 플레이 가능). settings에서 둘 다 자유 변경 가능
- **마우스 바인딩을 project.godot이 아닌 Main.gd에서**: project.godot의 InputEventMouseButton 직렬화 형식이 길고 오류 가능성이 큼. `_ensure_mouse_event`로 idempotent 추가가 더 안전. 사용자가 명시적으로 제거하면(v2 cfg에 없으면) 다음 부팅에 마우스 사라짐 — Main이 다시 추가하지 않도록 조건 검사
- **WASD를 ui_*에 추가하되 화살표 유지**: action_add_event는 기존 이벤트를 보존하므로 둘 다 동작. 양손/한손 모두 OK. 게임 액션과 충돌하는가? Title/Pause/LevelUp 화면에서는 게임 입력이 비활성(혹은 paused)이라 안전. Stage 안에서는 메뉴 노출이 아닌 한 ui_left/right이 작용하지 않음
- **v1 cfg 키바인드 폐기 결정**: 마이그레이션 코드를 짜기보다 단순 폐기. v1에 저장된 사용자는 다음 진입 시 새 default(마우스 + WASD primary) 적용 + 다음 저장에서 v2 전환
- **mb.canceled 검사 추가**: 마우스 캡처 중 휠/드래그/우클릭 컨텍스트 메뉴 시도가 캡처되지 않도록 가드

### 디버깅 노트
- `Script inherits from native type 'StaticBody2D', so it can't be assigned to an object of type: 'Node2D'`: TutorialDummy를 StaticBody2D로 바꿨는데 Tutorial.gd가 여전히 `Node2D.new()` 후 set_script 했음. 네이티브 타입 불일치는 set_script가 거부함. `TutorialDummy.new()`(class_name 활용)로 즉시 해결
- Settings._input의 마우스 캡처 타이밍: 사용자가 키바인드 슬롯 버튼을 클릭 → press_down 이벤트 → _input 호출(이때 capturing_action=="" → return) → Button.pressed 시그널 → capturing_action 설정. 같은 클릭의 press_up은 not pressed라 캡처 안 됨. 다음 mouse 입력부터 정상 캡처. 별도의 grace timer 불필요
- WASD가 ui_*에 추가되면서 의도하지 않은 부작용 우려했으나, focus가 있는 Control에서만 작동하므로 게임 도중 영향 없음. Title/Pause/LevelUp 같은 메뉴 화면에서만 navigation으로 동작
