## 세션 2 — 난이도 강화 (적 2종 추가)

### 요약
세션 1에서 "다음 턴 우선"으로 잡힌 난이도 강화 작업 수행. 새 적 2종(자폭병/방패병) 추가, take_damage에 위치 정보 전달 경로 도입, 적 수 멀티플라이어 살짝 ↑, reset() HP 덮어쓰기 버그 수정.

### 주요 변경
- `Enemy.gd` — `BOMBER`/`SHIELD` enum, `BomberState(ROAMING/STALKING/ARMING)`, 각각의 _tick_ 함수
  - 자폭병: 50px/s 순찰 → 감지 시 1.4배 추격 → 90px 안에 들어오면 0.7s 점멸 → 반경 70 폭발(1뎀). HP 1
  - 방패병: 55px/s 순찰, 같은 높이대(180×60) 내에서 플레이어 방향으로 dir 고정. 정면 32px 안에서 0.8s 쿨다운 1뎀
  - `take_damage(amount: int, from_x: float = INF)` — 시그니처 확장. `_shield_blocks(from_x)`로 정면 판정, 막힘 시 `_show_block_spark` 노란 라인 표시
- `Bullet.gd` — `body.take_damage(damage, global_position.x)`로 자기 위치 전달
- `TutorialDummy.gd` — 호환을 위해 `take_damage(amount, _from_x: float = INF)` 추가
- `CharacterArt.gd` — `build_bomber`(28×40, 가슴에 빨간 폭탄 표식 + 십자), `build_shield`(28×40 + Shield 노드에 방패 plate/boss/edge)
- `Stage.gd._spawn_enemies` — `bomber`/`shield` 카운팅. 함수 단순화하고 RNG 시드는 사용 안 함(불필요).
  - 전투/근접전 태그 → patrol 6, bomber 1
  - 함정 태그 → bomber 1 보장
  - 전투 태그 → shield 1
  - stage ≥ 3 → bomber +1
  - stage ≥ 4 → shield 보장 + patrol +1
- `Stage.gd._spawn_enemy` — kind 매개변수 매핑 명확히(0=patrol, 1=sniper, 2=drone, 3=bomber, 4=shield)
- `BestiaryData.gd` — bomber/shield 도감 카드 추가
- `GameState.gd`
  - `enemy_count_multiplier`: 1=0.7→0.8, 2=1.0→1.1, 3=1.4→1.5
  - `reset()`에서 `player_max_hp = 3`/`player_hp = 3` 직후 `=5`로 덮어쓰던 버그 제거 (HP 3 정책 실제 반영)

### 결정 사항
- **자폭병 vs 자동 포탑**: 후보 4종 중 자폭병 선택 — 능동적 위협(접근하는 위협)이 게임에 새 패턴 추가. 자동 포탑은 저격수와 메커닉 겹침.
- **방패병 vs 천장 낙하물**: 방패병 선택 — 측면 사격/대시로 우회하는 능동 메커닉이 더 흥미. 천장 낙하물은 함정 확장이라 별개 작업으로.
- **take_damage 시그니처 확장 vs 별도 hit_from 메서드**: 시그니처에 `from_x: float = INF` 추가. 호출자(Bullet)와 다른 take_damage 구현체(TutorialDummy)는 옵셔널 처리. 별도 메서드는 두 경로 분기 부담만 늘어남.
- **자폭병 HP 1**: 멀리서 정리하면 안전, 근접 허용하면 회피 압박이 큰 비대칭 설계. 가까이 두면 위험, 멀리 두면 약하다는 명확한 가치.
- **방패병 HP 3 + 정면 무효**: 단순 탱커가 아니라 "정면 돌파 차단기" 역할. 플레이어가 dash/jump로 측면을 잡도록 강제(이 두 스킬은 STARTING_SKILLS).
- **enemy_count_multiplier 0.7→0.8 등 미세 상향**: 부스 환경에서 너무 빡빡하면 답답. 새 적 2종이 추가됐으니 카운트 자체는 보수적으로만 ↑.
