## 결정 사항
- **AI 컨셉 유지** — 외부 픽셀/벡터 자산 거부. 코드 도형 + 외곽선/디테일이 정체성. *Why:* 사용자 명시 "AI가 만들었다는 컨셉을 밀려면 네가 만드는 게 제일 낫다". *How:* 새 비주얼은 _filled/_filled_circle 헬퍼로
- **5두신** — 4두신 chibi / 8두신 사실적. 5가 게임 캐릭터다움 균형. *Why:* 사용자 "다리가 길고 상체가 없는 수준" 피드백
- **신뢰도 = trust − aggression** — 둘 다 높으면 neutral, 한쪽 우세일 때만 명확. *Why:* 둘이 독립적으로 결말에 영향, 단계는 차이로
- **prefix random 풀** — 같은 단계에서도 hover마다 다름. *Why:* 사용자 "다 똑같이 음… 이렇게 나와서 단조롭다"
- **가시 dmg 옵션** — sewers 우측만 dmg 2로 차등. 다른 함정 영향 없음
- **HUD 2줄 + 스킬 작게** — 한 줄에 다 넣으면 화면 밖. 스킬은 부차 정보. *How:* hb(18px) + hb2(14px)

- **가시 파이프 베이스** — 30px 빨간 띠 → 10px 어두운 금속. *Why:* sewers 우측 등 mid-air 가시가 base 없이 떠보였음. 파이프 형태가 "박혀 있다" 인상

### 결정 사항
- **체인 + 미니 플랫폼** — 가시 베이스가 mid-air라 구조 연결이 시각으로 필요. 와이어 160px가 "위에서 매달림" 인상의 핵심

## 세션 2 — Xbox 패드 지원 + 스토리 모드

### 요약
키보드 외 게임패드(Xbox) 지원 추가. 동시에 키보드 조작이 어려운 사람을 위한 간략화된 "스토리 모드" — 체력 무제한, 드론 배제, 보스 단일 페이즈, 5스테이지로 압축.

### 주요 변경
- `project.godot` — 모든 액션에 InputEventJoypadButton/Motion 매핑 (좌스틱·D-Pad 이동, A 점프/확정, X 사격, B 대시, Y 스킬, START 메뉴, RT/RB 보조)
- `scripts/GameState.gd` — save/load_settings에 joy_button/joy_motion 직렬화. **SETTINGS_VERSION 2→3** 으로 올려 기존 cfg가 InputMap을 덮어써 패드 매핑을 지우는 버그 우회. story_mode 플래그 + effective_total_stages() helper. damage_player 무력화 분기.
- `scripts/Settings.gd` — 키 라벨에 패드 버튼/스틱/트리거 한글 표기. 키 리매핑 시 조이패드 이벤트 보존. 기본값 리셋에 패드 매핑 포함. Xbox 매핑 안내 한 줄 추가.
- `scripts/Title.gd` + `scenes/title.tscn` — "스토리 모드" 버튼 추가. 첫 방향 입력 시 StoryButton에 포커스 잡고 D-Pad/스틱으로 네비, A로 활성화. 그 전엔 SPACE/A로 normal-mode 즉시 시작.
- `scripts/Stage.gd` — _spawn_from_enemies_dict가 story_mode일 때 drone 스킵. HUD/하든 archive에서 effective_total_stages 사용.
- `scripts/BossSentinel.gd` — story_simplified 플래그. true면 P2/P3 전환·잔당 소환 모두 생략, HP 24→8.
- `scripts/RouteData.gd` — STORY_SCHEDULE 사전으로 5스테이지 고정 풀 (back_alley/rooftops → subway/watchtower → ward/sewers → escape → lab). 도전·???·드론 맵 모두 제외.
- `scripts/Briefing.gd` / `scripts/RouteMap.gd` — TOTAL_STAGES → effective_total_stages()

### 결정 사항
- **패드 매핑은 project.godot에 박고, Settings는 보존만** — 사용자가 키만 리매핑할 때 패드가 지워지지 않게 _input과 _apply_default_keybindings 양쪽에서 분리 처리. *Why:* MAX_KEYS_PER_ACTION 늘리면 UI 복잡 → 패드는 별도 채널로.
- **SETTINGS_VERSION 버전업으로 마이그레이션** — 기존 v2 cfg는 input 섹션 무시 → project.godot 기본값 유지. *Why:* "패드가 인게임에서만 안 됨" 버그가 정확히 이것 때문 (Title은 cfg 저장 후 리로드 전이라 InputMap에 패드 살아있음).
- **Title 첫 방향 입력 트리거 포커스** — 키보드/마우스 사용자 흐름(SPACE 누르면 시작) 보존하면서 패드 사용자가 D-Pad로 버튼 선택 가능. *Why:* 자동 포커스 잡으면 A 누름 = 즉시 버튼 활성화 → "그냥 시작" 흐름 깨짐.
- **스토리 모드는 5스테이지 고정 스케줄** — 풀 셔플·도전·hidden 모두 배제. *Why:* "필수적인 부분만". 보스 도달까지 짧고 명확한 동선이 우선.
- **story_mode는 영속화 안 함** — Title 버튼으로만 켜고 reset()이 끄는 휘발성. *Why:* 모드 토글이 cfg에 박히면 다음 세션에 의도치 않게 켜진 상태로 시작 가능.

### 미해결
