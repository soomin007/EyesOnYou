## 결정 사항
- **결말 C followup 진짜 원인**: 디버그 print 로그 (`[Ending] _start_line idx=2/2`)가
  결정적. choice 라인에서도 `silent_timer += delta`가 누적되어 1.5초 뒤 line_idx가
  진행됨 → `_on_sequence_done()` 호출되며 sequence_complete=true 박힘 → 그 후
  `_pick_choice`가 와도 `_process`는 hold-to-quit 분기로만 가서 typing 안 돌림.
  여러 세션에 걸쳐 watchdog·lockout 등으로 우회하려다 실패한 이유는 원인이 다른
  곳이었기 때문. **로그 안 박았으면 못 잡음.**
- **lore_label 전체 제거 vs 톤다운**: 사용자 "다른 방법 없어?" 워딩에 따라 제거.
  격리병동은 같은 맵에 ARCTURUS 문서 이스터에그가 있어 기능 중복. 핵심부/탈출로는
  ambience(격자/형광등) 자체로 분위기 충분.
- **자막 큐 `clear()` 위치**: `_start_arcturus_sequence` 진입 시점. 이스터에그가
  paused=true로 전환되면 SceneTreeTimer가 멈췄다가 풀린 뒤 한꺼번에 흐르며 outro와
  겹치는 현상 차단.
- **탈출로 cross-fade 구간**: 진행률 0.40~0.75. 너무 빨리 시작하면 터널 진입부
  분위기 안 살고, 너무 늦으면 도시 야경이 짧게만 보임. 35% 폭이 자연스러움.
- **CanvasLayer는 부모 modulate 안 받음**: 처음에 vignette을 CanvasLayer로 만들어
  cross-fade 안 됨. 비네트 자체 빼고 형광등+안개로만 터널 표현 — 도시 전환되면
  좌우 어두움 자연스러움.
- **퍼즐 시스템 디자인 문서**: 코드 시작 전 한 번 정리. attack 키 재사용·튜토리얼
  비밀칸·ward 이스터에그 변경 등 결정 근거를 다음 세션에서도 확인 가능하게.

### 주요 변경
- `scripts/LeverInteractable.gd` 신규 — Area2D + body_entered/exited로
  player.nearby_lever 세팅. `try_pull()` → pulled(lever_id) signal emit.
  one_shot 기본(true)일 때 풀린 후 locked + nearby_lever 클리어.
  ARCTURUS 청색 hint glow + 손잡이 회전 애니메이션.
- `scripts/Player.gd` — `nearby_lever: Node` 변수 추가. `_physics_process`의
  attack 입력 분기 변경: 레버 영역 내에서는 just_pressed 시 try_pull,
  꾹 누름 사격은 차단. 영역 밖이면 기존 동작.
- `scripts/Stage.gd`
  - `_ready`에서 `_build_lever_puzzles()` 호출 추가
  - `_spawn_lever / _spawn_closed_hatch / _open_hatch / _spawn_drop_platform /
    _descend_drop_platform` 헬퍼
  - `_build_back_alley_secret`: 지면 (1300, 588) 레버 → (2300, 290) 해치 fade
    + (2150, 380) 강하 발판 + XP orb 5
  - `_build_rooftops_secret`: 시작부 (200, 3060) 레버 → (200, 2820) 환기구
    fade + 사다리 발판 2개 강하 + HP 1 + XP 2

### 결정 사항
- **attack 키 재사용 vs 새 interact 키**: 재사용 채택. 영역 안에서는 사격 완전
  차단(꾹 누름 포함) — 처음엔 just_pressed만 가로채고 elif로 사격 fallback
  했는데, 이미 누른 채 영역 진입하면 사격됨. 영역 안에서는 attack 입력을
  완전히 흡수하는 단순 분기로 통일.
- **드롭 발판 collision 초기 비활성**: invisible(modulate.a=0) 상태에서도
  StaticBody의 collision은 살아있음. 풀리기 전 플레이어가 안 보이는 발판
  위에 떠있는 버그 차단을 위해 `col.disabled = true`로 시작, descend 시 해제.
- **레버 layer/mask**: lever.collision_layer=0(어디에도 안 보임) +
  collision_mask=2(player layer 감지). Player.collision_layer=2 기존 값과 매칭.
- **MapData 확장 vs Stage 인라인**: 비밀칸은 맵별 1~2개라 MapData 스키마 확장
  비용이 큼. Stage에 직접 좌표/orb 배치를 하드코딩 (`_build_back_alley_secret`
  같은 per-route 함수). 향후 갯수 늘어나면 MapData 추가 검토.
- **하나의 commit**: 레버 시스템 + 두 맵 비밀칸을 하나로 묶음. 시스템 자체가
  비밀칸 없이는 dead code, 비밀칸도 시스템 없이는 안 동작 — 분리 의미 없음.
