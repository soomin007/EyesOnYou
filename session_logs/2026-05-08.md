Suno로 생성한 5트랙 BGM을 게임 흐름에 매핑(타이틀/튜토리얼·외곽·시설·보스·???)·
크레딧 화면 신설(엔딩 자동 진입 + 설정 탭에서 보기). 튜토리얼 키보드 모드의 키캡이

## 결정 사항
- 라우트 → 트랙 매핑은 stage_index가 아니라 route_id로. 같은 stage에 있어도 외곽
  통로(early)와 시설 내부(mid_late)는 톤이 달라야 한다고 판단. 사용자가 BPM 점진
  증가 의도를 명시했고, 시설 내부로 들어가는 순간이 자연스러운 BPM step-up 지점.
- 크레딧은 두 진입점 공통 화면. scene 모드에선 종료 시 TITLE로(`GameState.reset`),
  overlay 모드는 그 자리에서 `closed` 시그널만 emit. 이렇게 두면 ESC 한 번으로
  설정 탭에서도 즉시 닫힌다.
- 설정의 마스터 볼륨은 BGM에 즉시 반영, SFX는 placeholder(SFX 신호 연결 추후).
- 키캡 수정은 `_make_keycap`에 `allow_pad_style` 옵션을 추가하는 방향. 글자 텍스트
  자체가 "A"라는 사실에 의존해 색을 정하는 기존 로직은 키보드 A/B/D-pad 글자
  ("←") 등을 구분 못 하므로 호출자가 모드 정보를 같이 넘기는 게 옳다.
- 사격 표지의 마우스 픽토그램은 모드 swap에 같이 들어가도록 hbox 빌드 단계에서
  prepend. 이전에 `_make_attack_sign`(키보드 전용 정적 버전)이 따로 있던 흔적은
  legacy로 남기고, 새 표지는 `_make_attack_sign_dynamic` 사용.

## 세션 2 — 레버 시스템 확장 + 도전방 연출

### 요약
puzzle_ideas 미해결 두 건(ward 이스터에그, datacenter 가시 비활성)과 도전방
(blackout) 구조 미비를 한 번에 처리. 가시가 지면 안에 묻혀 있던 버그 수정,
도전 진입 시 사이렌·암전·클리어 조건 배너로 "이게 도전이다" 통보.

### 주요 변경
- `scripts/PressurePlate.gd` 신설 — Area2D 기반 발판. require_armed 옵션으로
  "레버를 먼저 당겨야 활성" 시퀀스 가능. one_shot=true 기본.
- `scripts/Stage.gd`
  - `_spawn_toggleable_spike` + `_set_spike_group_active` — 시각/콜리전을
    Node2D wrapper로 묶어 한 번에 토글. _build_spike 자체는 그대로 두고,
    이 helper가 wrapper를 만든 뒤 _build_spike의 자식들을 reparent.
  - **ward**: `_tick_arcturus`/`_update_arcturus_indicator` 제거. 잠긴 문 앞에
    require_armed 발판, 맨 끝 상층(2900, 388)에 레버. 레버 → 발판 arm,
    발판 step → ARCTURUS 시퀀스. 5초 hold 변수(`arcturus_hold_t` 등) 정리.
  - **datacenter**: `_build_datacenter_secret` 신설. 지면(y=814)에 토글 가시
    2구간(x=550, x=1500), 상층 우측(1200, 320)에 레버. 당기면 양 구간 동시
    비활성 + VEIL "전기가 끊겼어요" 한 줄.
  - **blackout**: `_setup_challenge_mode`가 즉시 활성 → `challenge_pending=true`만
    세팅. `_build_challenge_gate`로 입구 폴리스 라인 줄무늬 문 + 발판 빌드.
    발판 step → `_start_challenge_run`이 문 fade + 사이렌 플래시 2회 +
    challenge_dark_layer fade-in + 타이머 HUD + "BLACKOUT RUN / N초 안에 골 도달"
    배너. 진입 사전 VEIL 경고도 한 줄.
- `scripts/MapData.gd` blackout 가시 좌표 수정 — y=660(지면에 묻힘) → y=594
  (지면 윗면). x도 발판 사이 갭에 정렬(480/950/1500/2050).

### 결정 사항
- ward 레버 위치는 잠긴 문에서 멀리 떨어진 상층 끝(x=2900) — 의도적 backtracking.
  "발판→레버 발견→되돌아와서 밟기" 두 단계 능동 행동이 1회용 의식적 진입에 가깝다.
  바로 옆에 두면 의식 없이 자동 발견되어 의미가 줄어든다.
- datacenter는 ARENA(전투 클리어)인데 가시를 깔면 patrol 적이 같이 죽을 수 있어
  _on_spike_touched가 player 한정으로 가드되어 있는지 확인 — collision_mask=2(player)
  로 충돌 자체가 적과 일어나지 않음. 안전.
- 도전방 입구를 발판으로 한 이유: 레버는 "당긴다"는 명시적 행동이지만, 발판은
  "들어선다"는 의사 표현. 도전 시작은 후자의 톤이 더 자연 — 무거운 결정을 가벼운
  스텝으로. + 사용자 요구: "안이 안 보이는 벽으로 가려진 곳 앞에 가서 발판이나
  레버를 작동시키면 사이렌이 울리면서 불이 꺼지고".
- 사이렌은 SFX가 아직 없어 화면 빨강 플래시 2회로 시각화. 나중에 SFX 들어오면
  같은 timing으로 `[siren_loop_short]` 재생만 추가하면 됨.
