# 2026-05-16 세션 로그

## 요약
설계 doc(`docs/design/sfx_list.md`)과 코드 사이 불일치 발견: doc은 Patrol을 "정찰병 사격(ranged shooter)"으로 정의했지만 `Enemy.gd`는 melee charge로만 구현됨. 사용자 결정으로 doc 쪽에 맞춰 ranged 공격을 추가 — Patrol이 중거리에서 사격, 근접에서 돌진하는 이중 모드가 됨.

## 주요 변경
- 신규 `scripts/EnemyBullet.gd` — 적 발사용 프로젝타일 (Area2D, 360px/s, 벽+플레이어 충돌, 주황 트레일)
- `scripts/Enemy.gd`:
  - `PatrolState` enum에 `FIRING` 추가
  - 거리 상수 `PATROL_CHARGE_RANGE=120` (이내면 돌진, 초과~DETECT 안이면 사격)
  - `PATROL_FIRE_INTERVAL=1.5`, `PATROL_FIRE_AIM_TIME=0.3`, `_patrol_fire_interval()` (Risk 3 ×0.7 보정)
  - `patrol_fire_armed: bool` — FIRING 내부에서 조준/쿨다운 phase 구분
  - `_tick_patrol` ROAMING 진입 분기: 거리 보고 TELEGRAPH 또는 FIRING으로 갈림
  - 신규 `_patrol_fire(p)` — EnemyBullet spawn + `enemy_patrol_fire` SFX 재생
  - 기존 `SfxPlayer.play("enemy_patrol_fire")` TELEGRAPH→CHARGING 위치에서 제거 (실제 사격 시점으로 이동)
- `docs/SPEC.md` Patrol 항목에 사격 phase 명세 추가
- `scripts/BestiaryData.gd` patrol blurb에 "중거리 사격" 단서 추가

## 결정 사항
- **사격 + 돌진 이중 모드 채택**: 사격 전용으로 바꾸면 정찰병 정체성(압박감) 사라지고 sniper와 역할 겹침. 중거리에서 압박하고 근접하면 돌진하는 게 가장 distinct.
- **hitscan이 아니라 실제 프로젝타일**: 회피 여지를 주려는 의도. Player bullet 900px/s 대비 EnemyBullet 360px/s라 점프/대시로 충분히 피할 수 있음.
- **EnemyBullet 별도 클래스 (vs Bullet.gd 분기 추가)**: Bullet에 enemy flag 추가하면 hit_enemies·pierce·tracking·multishot 등이 의미 없는 멤버로 남아 지저분. 적 전용 60줄 클래스가 더 읽힘.
- **사격/돌진 텔레그래프 색 구분**: 사격(노랑 점멸) vs 돌진(빨강 점멸). 플레이어가 시각만 보고 어떤 공격 오는지 알 수 있게.
- **`PATROL_CHARGE_RANGE=120`은 경험치**: detect_x=260의 절반. 플레이테스트로 조정 가능.

## 미해결 / 다음 할 일

## 세션 2 — 플레이테스트 피드백 정리

### 요약
첫 실 플레이에서 발견된 SFX 볼륨 불균형, Patrol 밸런스, Veil 대사 부정확, 보스 SFX 미연결을 한꺼번에 정리. sfx_list.md를 정확한 트리거·상태·ElevenLabs 프롬프트 포함한 단일 source로 재작성.

### 주요 변경
- **Patrol 밸런스** (`scripts/Enemy.gd` + `scripts/EnemyBullet.gd`):
  - `PATROL_CHARGE_RANGE` 120 → 240 (DETECT 260의 92% — 거의 항상 돌진, 사격은 가끔)
  - `EnemyBullet.BASE_SPEED` 360 → 240 (회피 가능 영역으로), `BASE_LIFETIME` 1.1 → 1.6 (사거리 유지)
- **SFX 볼륨 보정** (`scripts/SfxPlayer.gd::VOLUME_OFFSETS`):
  - `bullet_fire: -8`, `bullet_impact_wall: -5`, `enemy_hurt: -4`, `enemy_drone_drop: -8` (전부 너무 큼)
  - `bomb_throw: +4`, `bomb_explode: +6`, `enemy_drone_hover: +6` (거의 안 들림)
- **맵 경계벽 무음 처리**: `Stage.gd::_build_wall`에 `boundary_wall` 그룹 추가, `Bullet.gd::_on_body_entered`에서 그룹 체크해 SFX 생략. 게임 내 "실제 벽"이 아니라 가드라 impact 사운드 부적절.
- **Veil 대사 정확성 점검** (`scripts/VeilDialogue.gd`):
  - stage 2 "이 다음부터 저격수가 나와요" → "저격수 자리가 더 많아져요" (snipers는 stage 0 rooftops부터 이미 등장 가능)
  - stage 4 "드론도 섞여서 나와요" → "위에서 오는 것들 조심해요" (drones는 stage 3 cooling부터 등장 가능)
  - stage 0 "외곽 경비 두 명이에요" → "외곽 경비예요" (실제 패트롤 수는 3마리)
- **Boss SFX wire-up** (`scripts/BossSentinel.gd`):
  - `_transition_to` → `boss_phase_change`
  - `_arm_self_destruct` → `boss_self_destruct_alarm`
  - `_die` → `boss_death` + (자폭 카운트다운 중 처치 시) `boss_self_destruct_disarm`
  - `take_damage` (hp > 0) → `boss_hurt`
  - `_fire_missiles` → `boss_missile_launch`
- **sfx_list.md 재작성**: 단일 source로 통합. 각 SFX에 트리거 함수·길이·현재 보정·우선순위·상태·ElevenLabs 영문 prompt 포함. `boss_charge_telegraph`/`boss_charge_dash`는 코드와 불일치라 제거 (BossSentinel은 charge 공격 없음).
- **sfx_elevenlabs_prompts.md를 stub로 축소**: sfx_list.md 참조하라는 짧은 안내로 교체 (중복 sync 부담 제거).
- **`SfxPlayer.gd::KNOWN_SFX`**: 존재하지 않는 `boss_charge_telegraph`, `boss_charge_dash` 제거.

### 결정 사항
- **CHARGE_RANGE 240 (DETECT 260에 거의 닿음)**: 사용자가 "어지간하면 돌진하는 애처럼" 원함. 사격이 보조 행동 — 240~260 좁은 윈도우에서만 발사. 정찰병 정체성을 "사격"이 아닌 "압박"으로 재정의.
- **EnemyBullet 속도 240px/s**: Player bullet 900 대비 27%. 점프/대시로 충분히 회피 가능. lifetime을 1.6으로 늘려 사거리 ~384px 유지.
- **경계벽 그룹 마크 vs collision_layer 분리**: 별도 layer 만들면 enemy/bullet/player mask 도 다 손봐야 함. 그룹 마크가 1줄로 끝나 채택.
- **Veil 대사 일반화**: 특정 적 타입을 명시하는 멘트는 route 선택에 따라 거짓이 될 수 있음 → "저격수 자리가 많아져요" 같은 정도/방향 표현으로 변경.
- **boss_self_destruct_disarm 분기 조건**: `self_destruct_active and self_destruct_t < SELF_DESTRUCT_TIME` — 자폭 폭발 후 `_die`가 호출되는 경로(`_detonate`)에서는 `self_destruct_t >= SELF_DESTRUCT_TIME`이라 자동 분기됨.
- **sfx_list.md 단일 source 통합**: ElevenLabs prompts와 SFX list 두 파일 sync 비용 누적. 한 파일에 prompt 포함하고 별도 파일은 stub로.

## 세션 3 — 보스 폭탄 분리 + 드론 호버 positional loop

### 요약
사용자 피드백: 보스 폭탄 SFX가 드론 거 재사용해서 시끄럽고, 드론 호버는 거리에 따라 자연스럽게 들렸으면 좋겠다. 현재 SfxPlayer는 non-positional `AudioStreamPlayer` 풀이라 자동 거리 감쇠 없음 — 호버만 `AudioStreamPlayer2D`로 별도 처리.

### 주요 변경
- **보스/드론 폭탄 SFX 분리**:
  - `Bomb.gd::_ready`에서 `SfxPlayer.play("bomb_throw")` 제거 (자동 재생 안 함)
  - `BossSentinel.gd::_drop_bomb`에 `bomb_throw` 호출 추가
  - 드론은 기존 `enemy_drone_drop`만 재생 (음향 겹침 제거)
  - `SfxPlayer.VOLUME_OFFSETS["bomb_throw"]` +4 → −2
- **드론 호버 positional loop** (`scripts/Enemy.gd`):
  - 신규 상수: `DRONE_HOVER_VOLUME_DB=-14`, `DRONE_HOVER_MAX_DIST=900`, `DRONE_HOVER_ATTENUATION=1.6`
  - 신규 `hover_audio: AudioStreamPlayer2D` 필드, `_setup_drone_hover_audio()`, `_sync_hover_audio_volume()`
  - 드론 spawn(`_ready` DRONE 분기) 시 audio 노드 attach + autoplay, `_tick_drone`에서 매 tick 슬라이더 동기화
  - 기존 단발 hover SFX 호출 + `drone_was_hovering` flag 제거
  - `stream.duplicate()` 후 `loop=true` — 여러 드론이 같은 stream 인스턴스 공유 방지
- **`SfxPlayer.VOLUME_OFFSETS`**: `enemy_drone_hover` 항목 제거 (이제 SfxPlayer 경유 안 함). KNOWN_SFX entry는 유지(register만 됨, 호출 없음).
- **sfx_list.md 갱신**: `bomb_throw`를 "보스 전용"으로 명시, `enemy_drone_hover`를 "positional AudioStreamPlayer2D"로 표기 + ElevenLabs prompt 강화(여러 드론 누적 안 시끄럽게).

### 결정 사항
- **`Bomb._ready`에서 자동 재생 안 함**: spawner에 책임 위임. 드론은 enemy_drone_drop, 보스는 bomb_throw. 중복 layering 없음, 음향 캐릭터 분리.
- **호버만 positional**: 전체 SFX positional로 만들면 큰 리팩토링 + AudioStreamPlayer2D 풀 관리 비용. 호버처럼 "거리 감쇠가 정체성"인 SFX만 개별 처리.
- **`stream.duplicate()` 필요**: AudioStreamMP3.loop는 stream 인스턴스 프로퍼티 — 공유 시 다른 드론 인스턴스의 loop 설정이 영향. 안전하게 복제.
- **`max_distance=900`은 화면 폭의 약 1.4배**: 화면 밖 드론도 멀리서 살짝 들림 — "위쪽 봐요" 직관 강화. 너무 크면 화면 안에 안 보이는 드론 소리에 헷갈림.
- **base volume `-14dB`**: 가까이서도 muted. linear_to_db(1.0)=0이라 슬라이더 풀에서도 -14가 base. 누적 가능성(여러 드론) 고려.
- **`bomb_throw` -2dB**: 보스 전용이 됐고 보스 폭탄 SFX는 "묵직하지만 너무 크면 안 됨". 추후 추가 보정 가능.
