# 2026-06-09 세션 로그

## 요약
backlog "못 잡는 적 안내"(사용자 요청) 구현. 측면 단독 둥지 저격수(회피 전용)를 VEIL이
"정면으론 못 잡으니 사선 피하거나 글라이드로 덮쳐라"로 명시하고, ACT3 시야 붕괴 후속 맵에선
함정/매복을 마커로 못 짚어준다고 진입 직후 미리 경고. 기존 트랩 "파괴 불가" 안내 패턴 재사용.

## 주요 변경
- `scripts/MapData.gd` — rooftops·watchtower·cooling에 `"nest_snipers": true` 플래그 추가
  (이 맵들은 저격수가 전부 측면 단독 둥지 = 회피 전용). subway(지붕 본경로)·datacenter(웨이브)는 미부착.
- `scripts/Stage.gd`
  - `_spawn_enemy` — `nest_snipers` 맵의 저격수(kind 1)에 `avoid_only` 메타 태깅.
  - `_tick_avoid_warning`(신규, `_process`에 추가) — `avoid_only` 적에 430px 근접 시 1회 VEIL 안내.
    평시: "정면으론 안 닿아요. 사선 피하거나 글라이드로 위에서 덮쳐요"(글라이드-저격 상성 학습).
    붕괴 시: "잘 안 보여요. 사선 피하든지 글라이드로 덮쳐요"(톤 변형).
  - `_tick_trap_warning` — `veil_degraded`면 "함정이 있는 것 같아요, 잘 안 보여요" 붕괴 톤으로 분기.
  - `_arm_degraded_hazard_warning`(신규, `_setup_veil_mistakes`에서 호출) — 이미 붕괴 상태로 진입한
    ACT3 후속 맵이 함정/둥지를 가지면, 진입 멘트 가신 뒤(6s) "여기는 잘 못 봐요, 직접 살펴요" 미리 경고.
- `docs/design/backlog.md` — "못 잡는 적 안내" [x] 완료 처리(구현 요약 + 남은 플레이테스트 항목).

## 결정 사항
- **회피 전용 식별 = 맵 레벨 플래그**(`nest_snipers`)로. 위치 리스트 복제(좌표 매칭 깨지기 쉬움) 대신
  "이 맵의 저격수는 전부 둥지"라는 사실을 boolean으로 — rooftops/watchtower/cooling 모두 해당.
- **안내 문구는 "못 잡는다"가 아니라 "정면으론 못 잡으니 글라이드로"** — 둥지 저격수는 글라이드로
  도달·처치 가능(설계 의도: 회피든 처치든 선택). 그래서 글라이드-저격 상성을 가르치는 멘트로.
- **ACT3 미리 경고는 "처음부터 붕괴 상태로 진입한 맵"에서만** — 같은 맵에서 막 붕괴가 시작되는
  케이스는 `_fire_act3_vision` 자막이 이미 비트를 잡으므로 중복 방지. `veil_degraded`(맵 간 carry) 활용.
- **트랩 근접 경고도 붕괴 톤 변형** — 마커가 무너진 ACT3에선 "파괴 불가" 단언 대신 "잘 안 보인다"로.

## 미해결 / 다음 할 일

  멘트와 안 겹치는지 ③ 글라이드 유도 멘트가 실제로 글라이드 사용을 유도하는지.

  hop 가능. subway(천장 낮음)·ARENA·escape(브리더)는 부적합/보류. 220px = ward와 동일(검증된 값).

## 세션 3 — 플레이테스트 피드백: 크래시 2건 + 폴리시 + UI + 연습장 + 문서

### 요약
사용자 플레이 중 보고된 크래시 2건과 체감 문제(포탑 무위협·오브 흡인)를 수정하고, 부활/방어막
용어 통일 + 쿨타임 가독성, 연습장 개선, README 재작성·known_issues 기록까지 자율 진행.

### 주요 변경
- **크래시 2건** (d7782ca)
  - `RouteMap.gd`: waves 적 종류 집계에서 `int(위치배열)` 호출 크래시(datacenter 카드) → `arr.size()`.
  - `Enemy.gd`: 사망 후 씬 전환 중 트리 밖 노드의 `_find_player` → `get_tree()` null 크래시 가드
    (+ `_physics_process` `is_inside_tree` 가드).
- **글라이드 게이트 오브 흡인** (d7782ca) — `ExpOrb.attract_range` 변수화 + `gate_orbs` 데이터 키.
  게이트 보상은 60px 흡인(기본 220)으로, 아래/옆 메인 경로에서 바닥 뚫고 빨려오던 것 차단.
  ward/cooling/rooftops/watchtower 게이트 오브를 `gate_orbs`로 이동.
- **포탑 위치** (d7782ca) — 발판 top과 같은 높이라 무위협이던 가로 포탑을 watchtower는 갭(점프 경로)
  높이(1440/2060), datacenter는 랙 위 body 높이(550)로.
- **부활/방어막 용어 + 쿨타임 UI** (5256562) — `SkillTreeData` shield 라인 "방어막"→"부활"(비상 부활/
  부활 회복+/부활 재충전). HUD에 "부활" 슬롯 추가(T3 재충전 남은 초), 에너지 방어막 게이지 라벨에 실제
  남은 초 표시. `_arm_degraded_hazard_warning`을 `_ready`로 분리(연습장 토글로도 테스트 가능).
- **연습장 개선** (5256562) — 시야붕괴(veil_degraded) 토글 + 맵 선택 시 그 맵 기본 risk/reward/stage 자동설정.
- **문서** (7f50692) — README 재작성(스킬/위협/구조/연습장/진행상태), known_issues 4건, growth_system 용어 정합.

### 결정 사항
- **크래시 가드는 단일 길목에** — player 조회를 `_find_player` 하나로 모아 거기서 null 가드 →
  모든 호출처(_fire/_touch/_aim/_encounter) 동시 보호. 흩어진 가드보다 견고.
- **게이트 보상은 흡인 반경으로 게이팅** — 위치만으론 부족(수직 맵은 옆 발판이 220 안). 흡인 반경을
  "도달 의도"에 맞춰 줄이는 게 본질적 해법. known_issues에 기록.
- **가로 포탑은 발판 top 금지** — 갭/ body 높이로. 같은 높이면 탄이 표면을 스쳐 무해.
- **부활 vs 방어막 분리** — 같은 "방어막" 단어가 두 라인에 겹쳐 혼동(사용자 지적). shield=부활,
  barrier=에너지 방어막으로 고정. 둘 다 HUD에 남은 초 게이지.
