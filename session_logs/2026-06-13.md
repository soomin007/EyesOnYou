## 결정 사항
- **저격수 사거리는 '링', 포탑은 '선 길이 정합'으로 서로 다르게 처리.** 저격수는 투사체 없는
  히트스캔이라 위협이 *반경*(원) → 점선 링이 맞고, 포탑은 직선 탄이라 위협이 *선* → 선 길이를
  실제 사거리에 맞추는 게 맞다. 포탑은 선을 줄이지 않고 늘려 맞춤(watchtower 설계가 736px 가로탄
  도달을 전제로 하므로 총알을 줄이면 등반 압박이 깨짐).
- **포스터는 인엔진 + 실제 스크린샷.** 사용자 선택(인엔진 씬). AI 생성 일러스트는 게임 실제 화면과
  어긋나 "게임을 잘못 대변"하므로 배제. 키 비주얼은 BriefingVisual의 VEIL 눈(자산 의존 없는 _draw)
  재현 + Screenshotter로 찍은 실제 게임 화면.
- **스크린샷은 Stage를 자식 인스턴스화해 캡처.** change_scene는 하니스를 날려 캡처 불가 → 자식으로
  add_child 후 메인 뷰포트 캡처. GameState.record_route_choice로 라우트 세팅을 게임과 동일하게.
- **챗 버전 병행 지원.** 완전 일관성은 인엔진 PNG, 회화적 키 비주얼이 필요하면 챗 + brief 문서.

## 세션 2 — 사거리 정합 / VEIL 어투 아크 / 포스터 다양화

### 요약
저격수 인식 거리를 링과 정확히 일치시키고, 재발한 VEIL 대사 어미 반복(~게요 연발)을 고쳤다.
튜토리얼에 VEIL(눈+격식체 음성)을 도입해 분위기·인물·배경을 채웠고, 이를 "어투 아크"(초반 격식체→
후반 ~해요체)의 출발점으로 삼았다. 포스터 스크린샷을 액션 일변도에서 맵 선택·스킬 트리·전투로 다양화.

### 주요 변경
- `scripts/Enemy.gd` — 저격수 사거리 링 경계 1.15배→1.0배(실제 교전=표시 정합).
- `scripts/VeilSight.gd` — 인트로 "먼저 볼게요…짚어둘게요"(~게요 연발) → "먼저 봐요…짚어둘게요".
- `scripts/Tutorial.gd` — VEIL 눈(BriefingVisual 우상단) + 단계별 격식체 음성 자막(_veil_say).
  레벨업 카드 멘트도 격식체로. 어투 아크 출발점.
- `scripts/Screenshotter.gd` — 맵 선택(RouteMap)·스킬 트리(SkillTreeOverlay)·액션(적 근처 이동+다중사격)
  ·튜토리얼 캡처로 확장.
- `scripts/PosterCanvas.gd` — 스크린샷 3컷=맵 선택·스킬 트리·전투. VEIL 인용 교정.
- `poster_out/` — 포스터 + 스크린샷 갱신. memory: feedback_veil_dialogue_endings 추가.

### 결정 사항
- **VEIL 어투를 캐릭터 아크로(사용자 제안).** 개별 대사 땜질 대신 어투 자체를 서사 장치로:
  초반 격식체(~습니다, 차가운 관제사) → 후반 ~해요체(친근). 신뢰/의존 테마와 직결. 트리거는 진행도
  (ACT/stage) 기반 — "처음→마지막"이 시간 기반이므로 trust 점수와 무관하게 결정론적. 튜토리얼을
  가장 격식 있는 출발점으로 먼저 구현. **전체 롤아웃은 사용자 확인 후**(canon 영향 큼).
- **포스터는 액션만이 아니라 핵심 시스템도.** 맵 선택·스킬 트리가 게임 깊이를 더 잘 대변.
- **어미 반복 재발 → feedback 메모리화.** 같은 실수 두 번(06-11, 06-13)이라 세션 간 기록.
