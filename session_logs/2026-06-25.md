## 결정 사항
- **신뢰 임계 2/3(사용자 선택):** 과반 초과/2-3/4 중 "확실한 다수 2/3". 한 런 추천 5~6회 중 4번 이상
  따라야 신뢰. 공격축 임계(`SCORE_THRESHOLD=4`)는 피드백이 신뢰축만 지적해 유지.
- **데모 백포트(콘텐츠 동결 예외):** 엔딩 임계는 밸런스 결함 수정으로 간주해 데모(이 repo)에 적용. 본편
  (EyesOnYou-Full)은 막3+엔딩 재설계로 EndingResolver를 통째 교체할 예정이라 지금은 손 안 댐.
- **키 복원 "버그"의 정체 = 코드 아님:** 복원 로직은 처음부터 `save_settings` 호출 + 헤드리스 라운드트립
  PASS로 정상. GitHub Actions(`deploy.yml`)가 push마다 자동 배포해 06-24 키 수정도 이미 배포됨 →
  사용자가 본 건 브라우저 캐시/IndexedDB 옛 cfg. 코드 수정 대신 캐시 안내 + known_issues 기록으로 종결.
  (사용자 지시로 이 피드백 항목은 무시.)
- **패치노트 UI = STATE_MODE 패턴 재사용:** 좌측 버튼 목록 + 우측 패널이 이미 검증된 구조라 키보드/패드/
  마우스 일관. 단일 소스는 `GameInfo.PATCH_NOTES`, 항목 추가만 하면 UI가 자동 반영.
